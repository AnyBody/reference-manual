AnyKinMotion¶

Type

Creatable

Parent class

AnyKinDriver

Description¶

AnyKinMotion is a kinematic driver class that is basically identical to the general AnyKinDriver, except that all constraint reaction forces are switched off. This implies that this driver only specifies motion but not a mechanical actuator that produce the motion. In contrast, AnyKinDriver has reaction forces switched on, by default, implying that the object is an “infinitely strong” mechanical actuator that will produce the motion no matter the external loads.

AnyKinMotion specified motion must be mechanically (kinetically) generated by other objects in the model. In a musculoskeletal model, AnyKinMotion objects are used to specify motion that is produced by the muscles.

Examples¶

Here is simple code block showing how to use this class to drive the kneeflexion of a model.

AnyKinMotion KneeFlexionDriver = {
    DriverPos0 = {120} * pi/180; // Start Position
    DriverVel0 = {10} * pi/180; // Start velocity
    //DriverAcc0 = ; // An acceleration can also be specified if desired
    AnyKinMeasure &Knee = ..BodyModel.Interface.Right.KneeFlexion; // Pointer to the desired measure. More could be added.
};

Kinematic weighting functions, i.e., time-functions (AnyParamFun) which multiplied to the kinemtic measures before it is used as constraints. The total number of elements must match the dimension of the kinematic measure.
